Project: AI Customer Support & Persona EngineGoalTo implement a modular AI Persona system using the Vercel AI SDK and Supabase, enabling vendors to deploy customized, tool-capable AI assistants that can identify products, handle complex variations, and execute orders.Phase 1: Schema & Infrastructure (The "Soul" of the AI)Establish the data layer in Supabase to store AI configurations.Persona Management: Create a dedicated table for AI Personas. Fields should include:system_prompt: The core logic and personality instructions.greeting_message: The initial hook for customers.description_template: Preset "vibes" for vendors (e.g., "Professional Assistant," "Streetwear Expert").temperature/top_p: Tuning the AI's creativity versus factual accuracy.Knowledge Base Placeholder: Ensure the schema is extensible for the Q&A/Knowledge Base coming in the next iteration.Type Generation: Use the Supabase CLI to generate TypeScript types for these configurations to keep the Vercel AI SDK implementation type-safe.Phase 2: The Vercel AI SDK ImplementationIntegrate the AI engine with the marketplace front-end.Multi-Persona Routing: Build a dynamic route that fetches the specific persona based on the vendor_id.System Prompt Assembly: Logic to merge the core Moxxa Mart rules with the vendor's specific "Extra Instructions" and personality.Multimodal Integration: Enable image support so the AI can "see" customer-uploaded photos to identify features or recommend matching items from the shared product gallery.Phase 3: Tool-Calling & Logic (The "Hands" of the AI)Develop the functional bridge between natural language and your database.search_products: A tool that queries the catalog using pgvector to find items based on descriptions or features.get_product_details: Must handle the Attribute-based variation system, pulling specific sizes, colors, and the corresponding "Variation Image Overrides".check_discounts: Logic to verify active promotions or bulk pricing for the AI to mention during the pitch.take_order: A multi-step tool that:Gathers necessary customer info (Name, Delivery, Phone).Asks for explicit confirmation before creating the order.Hands off the session to the Proof-of-Payment workflow.Phase 4: Generative UI & User ExperienceDefine an efficient way to render complex data in the chat.Server Components / Generative UI: Instead of the AI typing out long lists, use the Vercel AI SDK to return React Components (e.g., a horizontally scrollable Product Gallery or a "Confirm Order" card).Variation Pickers: If a product has multiple attributes, the AI should trigger a UI component that allows the user to select their size/color visually rather than typing it.Strategic ConsiderationsUsage Tracking: Implement a middleware to log tokens internally for cost analysis while tracking message counts for vendor billing tiers.Omnichannel Readiness: Ensure the tool-calling logic is decoupled from the UI so it can eventually support Telegram and Viber via the UMA (Unified Messaging Architecture).
