## EXTRACTION RULES

### Tables
- Use CREATE TABLE statements
- Include all columns, types, defaults
- Include constraints (PK, FK, UNIQUE, CHECK)

### Relationships
- Explicitly list all foreign keys separately for clarity

### Indexes
- Extract all indexes, including implicit ones if relevant

### Functions
- Extract ALL custom SQL functions
- Clearly identify which are used by triggers

### Triggers
- Extract trigger definitions
- Map each trigger → function → table → event (INSERT/UPDATE/DELETE)

### RLS Policies
- Extract ALL policies per table
- Include:
  - Policy name
  - Command (SELECT, INSERT, etc.)
  - USING and WITH CHECK expressions
- Group by table

### Storage
- List all buckets
- Extract access policies if available

## CLEANUP + ERROR HANDLING

You MUST handle broken or inconsistent state:

### Migration Issues
- Compare migration history vs actual schema
- Identify unapplied or partially applied migrations
- DO NOT blindly trust migration files

### Broken Triggers
- Detect triggers referencing missing functions
- Either:
  - Fix by linking to correct function
  - OR mark clearly as "INVALID" in output

### Policy Issues
- Remove duplicates
- Flag unsafe policies (e.g. USING true without auth)
- Ensure consistency with auth.uid() usage

### Output Strategy
- If something is broken:
  - DO NOT delete silently
  - Add comments like:

    -- WARNING: This trigger references a missing function
    -- TODO: Fix or remove
